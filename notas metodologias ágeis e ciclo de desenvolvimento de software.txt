Desenvolvimento de software: desenvolvimento iterativo na qual a ferramenta em evolução serve como meio de comunicação
Processo de software são metodologias para atividade, ações e tarefas para desenvolver um software
linear: cada estágio começa depois que a anterior finalizar
iterativo: processo linear cada estágio tem uma iteração entre as etapas
evolucionário: cada etapa vai evoluindo o projeto
paralelo: atividades ocorre de forma paralela e talvez não sequencial

Modelo prescritivo ou tradicional: ordem e consistência do processo
Modelo cascata: comunicação, planejamento, modelagem, construção e entrega. Linear, a proxima etapa começa só depois que a anterior finaliza, útil para requisitos definidaos e estáveis
Desvantagem: não lida com adaptação, longo tempo para a primeira versão, estado de bloqueio para equipe
Modelo V: Focado em testes

Modelo Incremental: fluxo cascata com pequenas versões do softwares entregues, requisitos bem definidos mas não refinados

Modelo evolucionário - Prototipação: bom para refinar requisitos, valida eficiência e interação com o usuário, pode evoluir ou ser descartável
Desvantagem: deixa de considerar a qualidade em prol da entrega rápida
Modelo evolucionário - Espiral: estratégia cíclica incremental com foco em diminuir riscos

Modelos especializados: necessita de métodos específicos de acordo com o contexto ex: métodos formais, orientado a aspectos
